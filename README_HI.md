## 📋 प्रोजेक्ट अवलोकन

**ForexSignalBot** Telegram के लिए एक उन्नत AI-आधारित सिस्टम है, जो Forex बाजार के लिए वास्तविक समय में सटीक ट्रेडिंग सिग्नल प्रदान करने के लिए डिज़ाइन किया गया है। **.NET 9** पर बनाया गया है और Clean Architecture और Domain-Driven Design सिद्धांतों का उपयोग करता है।

### 🌟 मुख्य विशेषताएं

- **🤖 AI सिग्नल जनरेशन:** उन्नत बाजार विश्लेषण एल्गोरिदम
- **📰 समाचार एकत्रीकरण:** 100 RSS फीड्स स्मार्ट वर्गीकरण के साथ
- **💬 Telegram में पूर्ण UI:** समृद्ध और इंटरैक्टिव उपयोगकर्ता अनुभव
- **🔗 स्वचालित फॉरवर्डिंग:** ट्रेडिंग प्लेटफॉर्म्स के साथ एकीकरण
- **🐳 Docker कंटेनराइजेशन:** आसान और सुसंगत तैनाती
- **🛡️ सुरक्षा:** मजबूत त्रुटि प्रबंधन और डेटा सुरक्षा

### 🏗️ सिस्टम आर्किटेक्चर

प्रोजेक्ट **Clean Architecture** और **Domain-Driven Design** सिद्धांतों पर बनाया गया है:

- **डोमेन लेयर:** मुख्य व्यावसायिक तर्क और इकाइयां
- **एप्लिकेशन लेयर:** एप्लिकेशन सेवाएं और use case ऑर्केस्ट्रेशन
- **इन्फ्रास्ट्रक्चर लेयर:** डेटाबेस और बाहरी API कार्यान्वयन
- **WebAPI:** REST API कंट्रोलर्स
- **TelegramPanel:** Telegram बॉट इंटरैक्शन प्रबंधन
- **BackgroundTasks:** Hangfire के साथ बैकग्राउंड टास्क प्रोसेसिंग

### 🛠️ तकनीकी स्टैक

- **.NET9 - मुख्य विकास प्लेटफॉर्म
- **PostgreSQL** - रिलेशनल डेटाबेस
- **Redis** - कैशिंग और बैकग्राउंड प्रोसेसिंग
- **Hangfire** - बैकग्राउंड टास्क प्रबंधन
- **Docker** - कंटेनराइजेशन और तैनाती
- **Entity Framework Core** - ORM
- **Polly** - त्रुटि प्रबंधन और लचीलापन

## 🚀 त्वरित प्रारंभ

आप इस प्रोजेक्ट को दो तरीकों से चला सकते हैं: Docker के साथ (त्वरित सेटअप के लिए अनुशंसित) या स्थानीय वातावरण को मैन्युअल रूप से कॉन्फ़िगर करके।

### विकल्प 1 Docker के साथ त्वरित प्रारंभ (अनुशंसित)

Docker के साथ मिनटों में पूरे एप्लिकेशन स्टैक — API, PostgreSQL डेटाबेस और Redis कैश — को चलाएं। **यह शुरू करने का सबसे तेज़ और आसान तरीका है।**

#### पूर्व आवश्यकताएं

*   **Docker Desktop**: सुनिश्चित करें कि यह आपके सिस्टम पर इंस्टॉल और चल रहा है। [यहां डाउनलोड करें](https://www.docker.com/products/docker-desktop/).

#### चरण 1रिपॉजिटरी क्लोन करें

अपना टर्मिनल खोलें और प्रोजेक्ट सोर्स कोड क्लोन करें।
```bash
git clone https://github.com/Opselon/ForexTradingBot.git
cd ForexTradingBot
```

#### चरण2: अपने रहस्य कॉन्फ़िगर करें

एप्लिकेशन को API कुंजी और पासवर्ड की आवश्यकता होती है। हम इसके लिए `.env` फ़ाइल का उपयोग करते हैं, जो निजी रखी जाती है।
1.  **पर्यावरण फ़ाइल बनाएं:**
    ```bash
    cp .env.example .env
    ```
2 **`.env` फ़ाइल संपादित करें:** नई `.env` फ़ाइल खोलें और अपने वास्तविक रहस्य मान भरें।
    *   `TELEGRAM_BOT_TOKEN`: इसे Telegram में `@BotFather` से प्राप्त करें।
    *   `POSTGRES_PASSWORD`: अपने डेटाबेस के लिए एक मजबूत, सुरक्षित पासवर्ड बनाएं।

#### चरण 3: एप्लिकेशन चलाएं! 🔥

Docker चलते हुए, प्रोजेक्ट रूट डायरेक्टरी से एक कमांड निष्पादित करें:
```bash
docker-compose up --build -d
```
यह कमांड API, PostgreSQL और Redis कंटेनर बनाता और शुरू करता है। API **शुरू होने पर डेटाबेस माइग्रेशन को स्वचालित रूप से लागू करने** के लिए कॉन्फ़िगर किया गया है।

#### चरण 4डेटाबेस पॉपुलेट करें
बॉट को RSS फीड्स की प्रारंभिक सूची की आवश्यकता होती है। DBeaver या DataGrip जैसे क्लाइंट का उपयोग करके डेटाबेस से कनेक्ट करें और `Populate_RssSources_Categories.sql` स्क्रिप्ट निष्पादित करें।
*   **होस्ट:** `localhost`
*   **पोर्ट:** `5432*   **डेटाबेस:** `forexsignalbot_db`
*   **उपयोगकर्ता:** `postgres`
*   **पासवर्ड:** `.env` में सेट किया गया `POSTGRES_PASSWORD`।

**🎉 बस इतना ही! आपका बॉट अब Docker के अंदर चल रहा है。**

### विकल्प 2: स्थानीय विकास सेटअप (Docker के बिना)

यदि आप एप्लिकेशन को सीधे अपने मशीन पर चलाना पसंद करते हैं तो इन चरणों का पालन करें।

#### पूर्व आवश्यकताएं
1**.NET9 SDK:**
    *   **.NET9 SDK (v9107 या नया)** इंस्टॉल करें।
    *   **डाउनलोड पेज:** [https://dotnet.microsoft.com/en-us/download/dotnet/90//dotnet.microsoft.com/en-us/download/dotnet/9*   `dotnet --version` चलाकर अपनी इंस्टॉलेशन सत्यापित करें।2**PostgreSQL डेटाबेस:**
    *   एक स्थानीय PostgreSQL सर्वर इंस्टॉल और चलाएं।
    *   एक डेटाबेस और उपयोगकर्ता बनाएं।
    *   `appsettings.Development.json` फ़ाइल में अपनी कनेक्शन स्ट्रिंग अपडेट करें।3 **Redis सर्वर:**
    *   Redis का उपयोग कैशिंग और बैकग्राउंड जॉब प्रोसेसिंग के लिए किया जाता है।
    *   **Windows के लिए:** **Memurai** जैसे Redis-संगत सर्वर इंस्टॉल करें।
        *   **इंस्टॉलेशन गाइड:** [https://docs.memurai.com/en/installation.html](https://docs.memurai.com/en/installation.html)
    *   **macOS/Linux के लिए:** पैकेज मैनेजर के माध्यम से इंस्टॉल करें (जैसे `brew install redis` या `sudo apt-get install redis-server`).

#### स्थानीय रूप से एप्लिकेशन चलाएं

उन डेवलपर्स के लिए जो एप्लिकेशन को सीधे अपने मशीन पर चलाना पसंद करते हैं, इन चरणों का पालन करें:
1  **रिपॉजिटरी क्लोन करें** (यदि अभी तक नहीं किया है)।
2.  **`appsettings.Development.json` कॉन्फ़िगर करें** अपनी स्थानीय डेटाबेस कनेक्शन स्ट्रिंग और अन्य सेटिंग्स के साथ।3  **डेटाबेस माइग्रेशन लागू करें:**
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```
4  **डेटाबेस पॉपुलेट करें** अपने स्थानीय डेटाबेस के खिलाफ `Populate_RssSources_Categories.sql` स्क्रिप्ट चलाकर।5.  **API चलाएं:**
    ```bash
    dotnet run --project WebApi
    ```

अधिक विस्तृत विवरण और प्रोडक्शन डिप्लॉयमेंट निर्देशों के लिए, कृपया [समर्पित INSTALL.md गाइड](https://github.com/Opselon/ForexTradingBot/blob/master/WebAPI/INSTALL.md) देखें।

## 🛠️ डेवलपर गाइड

इस खंड में विकास के लिए सामान्य कमांड शामिल हैं।

### डेटाबेस माइग्रेशन प्रबंधन

इन कमांड को चलाने से पहले, सुनिश्चित करें कि EF Core टूल्स इंस्टॉल हैं: `dotnet tool install --global dotnet-ef`

*   **नया माइग्रेशन जोड़ें:** जब आप डोमेन मॉडल बदलते हैं, तो एक नया माइग्रेशन बनाएं।
    ```bash
    dotnet ef migrations add YourMigrationName --startup-project WebApi --project Infrastructure
    ```
    *(`YourMigrationName` को वर्णनात्मक नाम से बदलें, जैसे `AddSignalStatus`)*

*   **माइग्रेशन लागू करें:** डेटाबेस स्कीमा को मैन्युअल रूप से अपडेट करने के लिए।
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```

### प्रोडक्शन बिल्ड बनाएं

एप्लिकेशन को डिप्लॉयमेंट के लिए स्वतंत्र निष्पादन योग्य फ़ाइल में कंपाइल करने के लिए:

```bash
# Windows x64 स्वतंत्र बिल्ड का उदाहरण
dotnet publish --configuration Release --runtime win-x64elf-contained true --project WebApi
```
*   आउटपुट `WebApi/bin/Release/net94ublish` फोल्डर में होगा।
