## 📋 Panoramica del progetto

**ForexSignalBot** è un sistema avanzato basato sull'IA per Telegram, progettato per fornire segnali di trading precisi in tempo reale per il mercato Forex. Costruito su **.NET9lizzando i principi dell'architettura pulita e Domain-Driven Design.

### 🌟 Caratteristiche principali

- **🤖 Generazione segnali IA:** Algoritmi avanzati di analisi del mercato
- **📰 Aggregazione notizie:**100ed RSS con categorizzazione intelligente
- **💬 UI completa Telegram:** Esperienza utente ricca e interattiva
- **🔗 Inoltro automatico:** Integrazione con piattaforme di trading
- **🐳 Containerizzazione Docker:** Distribuzione facile e consistente
- **🛡️ Sicurezza:** Gestione robusta degli errori e protezione dei dati

### 🏗️ Architettura del sistema

Il progetto è costruito sui principi di **architettura pulita** e **Domain-Driven Design**:

- **Livello Domain:** Logica di business centrale ed entità
- **Livello Application:** Servizi applicativi e orchestrazione dei casi d'uso
- **Livello Infrastructure:** Implementazione database e API esterne
- **WebAPI:** Controller REST API
- **TelegramPanel:** Gestione interazioni bot Telegram
- **BackgroundTasks:** Elaborazione task in background con Hangfire

### 🛠️ Stack tecnologico

- **.NET9aforma di sviluppo principale
- **PostgreSQL** - Database relazionale
- **Redis** - Cache ed elaborazione in background
- **Hangfire** - Gestione task in background
- **Docker** - Containerizzazione e distribuzione
- **Entity Framework Core** - ORM
- **Polly** - Gestione errori e resilienza

## 🚀 Avvio rapido

Puoi eseguire questo progetto in due modi: con Docker (raccomandato per la configurazione rapida) o configurando un ambiente locale manualmente.

### Opzione 1: Avvio rapido con Docker (Raccomandato)

Ottieni l'intero stack applicativo — API, database PostgreSQL e cache Redis — in esecuzione in minuti con Docker. **Questo è il modo più veloce e facile per iniziare.**

#### Prerequisiti

*   **Docker Desktop**: Assicurati che sia installato e in esecuzione sul tuo sistema. [Scaricalo qui](https://www.docker.com/products/docker-desktop/).

#### Passo 1: Clona il repository

Apri il terminale e clona il codice sorgente del progetto.
```bash
git clone https://github.com/Opselon/ForexTradingBot.git
cd ForexTradingBot
```

#### Passo2: Configura i tuoi segreti

Lapplicazione richiede chiavi API e password. Usiamo un file `.env` per questo, che viene mantenuto privato.

1a il file ambiente:**
    ```bash
    cp .env.example .env
    ```2**Modifica il file `.env`:** Apri il nuovo file `.env` e riempi i tuoi valori segreti reali.
    *   `TELEGRAM_BOT_TOKEN`: Ottieni questo da `@BotFather` su Telegram.
    *   `POSTGRES_PASSWORD`: Crea una password forte e sicura per il tuo database.

#### Passo 3gui l'applicazione! 🔥

Con Docker in esecuzione, esegui un singolo comando dalla directory root del progetto:
```bash
docker-compose up --build -d
```
Questo comando costruisce e avvia i container API, PostgreSQL e Redis. LAPI è configurata per **applicare automaticamente le migrazioni del database all'avvio**.

#### Passo 4: Popola il database
Il bot ha bisogno di una lista iniziale di feed RSS. Connettiti al database usando un client come DBeaver o DataGrip ed esegui lo script `Populate_RssSources_Categories.sql`.
*   **Host:** `localhost`
*   **Porta:** `5432   **Database:** `forexsignalbot_db`
*   **Utente:** `postgres`
*   **Password:** Il `POSTGRES_PASSWORD` che hai impostato in `.env`.

**🎉 È tutto! Il tuo bot ora è in esecuzione dentro Docker.**

### Opzione2gurazione sviluppo locale (Senza Docker)

Segui questi passi se preferisci eseguire l'applicazione direttamente sulla tua macchina.

#### Prerequisiti
1**.NET9 SDK:**
    *   Installa il **.NET9 SDK (v917 o successivo)**.
    *   **Pagina di download:** [https://dotnet.microsoft.com/en-us/download/dotnet/90//dotnet.microsoft.com/en-us/download/dotnet/9*   Verifica la tua installazione eseguendo `dotnet --version`.
2.  **Database PostgreSQL:**
    *   Installa ed esegui un server PostgreSQL locale.
    *   Crea un database e un utente.
    *   Aggiorna la stringa di connessione nel file `appsettings.Development.json`.
3.  **Server Redis:**
    *   Redis è usato per cache ed elaborazione job in background.
    *   **Per Windows:** Installa un server compatibile con Redis come **Memurai**.
        *   **Guida all'installazione:** [https://docs.memurai.com/en/installation.html](https://docs.memurai.com/en/installation.html)
    *   **Per macOS/Linux:** Installa tramite un gestore di pacchetti (es. `brew install redis` o `sudo apt-get install redis-server`).

#### Esegui l'applicazione localmente

Per sviluppatori che preferiscono eseguire l'applicazione direttamente sulla loro macchina, segui questi passi:
1  **Clona il repository** (se non lhai già fatto).2  **Configura `appsettings.Development.json`** con la tua stringa di connessione database locale e altre configurazioni.
3.  **Applica le migrazioni del database:**
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```
4  **Popola il database** eseguendo lo script `Populate_RssSources_Categories.sql` contro il tuo database locale.
5.  **Esegui l'API:**
    ```bash
    dotnet run --project WebApi
    ```

Per dettagli più completi e istruzioni di distribuzione in produzione, consulta la [guida INSTALL.md dedicata](https://github.com/Opselon/ForexTradingBot/blob/master/WebAPI/INSTALL.md).

## 🛠️ Guida sviluppatore

Questa sezione contiene comandi comuni per lo sviluppo.

### Gestisci migrazioni database

Prima di eseguire questi comandi, assicurati di avere gli strumenti EF Core installati: `dotnet tool install --global dotnet-ef`

*   **Aggiungi una nuova migrazione:** Quando cambi un modello di dominio, crea una nuova migrazione.
    ```bash
    dotnet ef migrations add YourMigrationName --startup-project WebApi --project Infrastructure
    ```
    *(Sostituisci `YourMigrationName` con un nome descrittivo, es. `AddSignalStatus`)*

*   **Applica migrazioni:** Per aggiornare manualmente lo schema del database.
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```

### Crea una build di produzione

Per compilare l'applicazione in un eseguibile autonomo per la distribuzione:

```bash
# Esempio per una build autonoma Windows x64
dotnet publish --configuration Release --runtime win-x64elf-contained true --project WebApi
```
*   Loutput sarà nella cartella `WebApi/bin/Release/net94ublish`.

### Passo 4: Wizard configurazione web ✨
Questo è il nuovo processo di configurazione semplificato.

1**Apri il pannello web:** Naviga verso **[http://localhost:50ogin.html](http://localhost:50 nel tuo browser.
2.  **Accedi:** Usa le credenziali predefinite:
    *   **Username:** `admin`
    *   **Password:** `admin`
    *(Ti verrà chiesto di cambiare questi dettagli sensibili durante il processo di configurazione iniziale nell'UI web per una migliore sicurezza.)*
3.  **Configurazione guidata:** Dopo aver effettuato l'accesso per la prima volta, verrai automaticamente reindirizzato a una pagina di configurazione sicura (`/indexapp.html`).
    *   In questa pagina ti verrà chiesto di inserire il tuo **Token Bot Telegram** e altre configurazioni principali.
    *   Il sistema **testerà in tempo reale** le tue credenziali per assicurarsi che siano valide prima del salvataggio.
    *   Una volta salvate, queste configurazioni vengono memorizzate in modo sicuro nel database, non in file di testo.
4.  **Popola il database:** Dopo la configurazione iniziale, ti verrà chiesto di popolare il database. Clicca sul pulsanteSeed Database" nell'UI web. Questo popolerà la lista iniziale di feed RSS e altri dati richiesti.

**🎉 È tutto! Il tuo bot ora è completamente configurato e in esecuzione.** Puoi gestire tutto dal pannello web.
