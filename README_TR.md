## 📋 Proje Genel Bakışı

**ForexSignalBot**, Forex pazarı için gerçek zamanlı kesin işlem sinyalleri sağlamak üzere tasarlanmış Telegram için gelişmiş yapay zeka tabanlı bir sistemdir. **.NET 9** üzerine inşa edilmiş ve Temiz Mimari ve Domain-Driven Design prensiplerini kullanmaktadır.

### 🌟 Temel Özellikler

- **🤖 Yapay Zeka Sinyal Üretimi:** Gelişmiş pazar analiz algoritmaları
- **📰 Haber Toplama:** 100+ RSS beslemesi akıllı kategorizasyon ile
- **💬 Telegramda Tam UI:** Zengin ve etkileşimli kullanıcı deneyimi
- **🔗 Otomatik İletim:** İşlem platformları ile entegrasyon
- **🐳 Docker Konteynerizasyonu:** Kolay ve tutarlı dağıtım
- **🛡️ Güvenlik:** Sağlam hata yönetimi ve veri koruması

### 🏗️ Sistem Mimarisi

Proje **Temiz Mimari** ve **Domain-Driven Design** prensipleri üzerine inşa edilmiştir:

- **Domain Katmanı:** Merkezi iş mantığı ve varlıklar
- **Uygulama Katmanı:** Uygulama servisleri ve use case orkestrasyonu
- **Altyapı Katmanı:** Veritabanı ve harici API implementasyonu
- **WebAPI:** REST API kontrolcüleri
- **TelegramPanel:** Telegram bot etkileşim yönetimi
- **BackgroundTasks:** Hangfire ile arka plan görev işleme

### 🛠️ Teknoloji Yığını

- **.NET9eliştirme platformu
- **PostgreSQL** - İlişkisel veritabanı
- **Redis** - Önbellek ve arka plan işleme
- **Hangfire** - Arka plan görev yönetimi
- **Docker** - Konteynerizasyon ve dağıtım
- **Entity Framework Core** - ORM
- **Polly** - Hata yönetimi ve dayanıklılık

## 🚀 Hızlı Başlangıç

Bu projeyi iki şekilde çalıştırabilirsiniz: Docker ile (hızlı kurulum için önerilen) veya yerel ortamı manuel olarak yapılandırarak.

### Seçenek 1: Docker ile Hızlı Başlangıç (Önerilen)

Tüm uygulama yığınını — API, PostgreSQL veritabanı ve Redis önbelleği — Docker ile dakikalar içinde çalışır hale getirin. **Bu başlamanın en hızlı ve kolay yoludur.**

#### Ön Gereksinimler

*   **Docker Desktop**: Sisteminizde kurulu ve çalışır durumda olduğundan emin olun. [Buradan indirin](https://www.docker.com/products/docker-desktop/).

#### Adım1: Depoyu Klonlayın

Terminalinizi açın ve proje kaynak kodunu klonlayın.
```bash
git clone https://github.com/Opselon/ForexTradingBot.git
cd ForexTradingBot
```

#### Adım2: Gizli Bilgilerinizi Yapılandırın

Uygulama API anahtarları ve şifreler gerektirir. Bunun için `.env` dosyası kullanıyoruz, bu dosya gizli tutulur.

1 **Ortam dosyasını oluşturun:**
    ```bash
    cp .env.example .env
    ```
2.  **`.env` dosyasını düzenleyin:** Yeni `.env` dosyasını açın ve gerçek gizli değerlerinizi doldurun.
    *   `TELEGRAM_BOT_TOKEN`: Bunu Telegramdaki `@BotFather`dan alın.
    *   `POSTGRES_PASSWORD`: Veritabanınız için güçlü, güvenli bir şifre oluşturun.

#### Adım3: Uygulamayı Çalıştırın! 🔥

Docker çalışır durumdayken, proje kök dizininden tek bir komut çalıştırın:
```bash
docker-compose up --build -d
```
Bu komut API, PostgreSQL ve Redis konteynerlerini oluşturur ve başlatır. API, **başlangıçta veritabanı migrasyonlarını otomatik olarak uygulamak** üzere yapılandırılmıştır.

#### Adım4 Veritabanını Doldurun
Botun RSS beslemelerinin başlangıç listesine ihtiyacı vardır. DBeaver veya DataGrip gibi bir istemci kullanarak veritabanına bağlanın ve `Populate_RssSources_Categories.sql` betiğini çalıştırın.
*   **Host:** `localhost`
*   **Port:** `5432 **Veritabanı:** `forexsignalbot_db`
*   **Kullanıcı:** `postgres`
*   **Şifre:** `.env`'de ayarladığınız `POSTGRES_PASSWORD`.

**🎉 İşte bu kadar! Botunuz artık Docker içinde çalışıyor.**

---

### Seçenek 2: Yerel Geliştirme Kurulumu (Docker Olmadan)

Uygulamayı doğrudan makinenizde çalıştırmayı tercih ediyorsanız bu adımları takip edin.

#### Ön Gereksinimler
1**.NET9 SDK:**
    *   **.NET9 SDK (v9107 veya daha yeni)** kurun.
    *   **İndirme sayfası:** [https://dotnet.microsoft.com/en-us/download/dotnet/90//dotnet.microsoft.com/en-us/download/dotnet/9*   Kurulumunuzu `dotnet --version` çalıştırarak doğrulayın.2**PostgreSQL Veritabanı:**
    *   Yerel bir PostgreSQL sunucusu kurun ve çalıştırın.
    *   Bir veritabanı ve kullanıcı oluşturun.
    *   `appsettings.Development.json` dosyasındaki bağlantı dizesini güncelleyin.
3.  **Redis Sunucusu:**
    *   Redis önbellek ve arka plan iş işleme için kullanılır.
    *   **Windows için:** **Memurai** gibi Redis uyumlu bir sunucu kurun.
        *   **Kurulum rehberi:** [https://docs.memurai.com/en/installation.html](https://docs.memurai.com/en/installation.html)
    *   **macOS/Linux için:** Paket yöneticisi aracılığıyla kurun (örn. `brew install redis` veya `sudo apt-get install redis-server`).

#### Uygulamayı Yerel Olarak Çalıştırma

Uygulamayı doğrudan makinenizde çalıştırmayı tercih eden geliştiriciler için bu adımları takip edin:
1  **Depoyu klonlayın** (eğer henüz yapmadıysanız).
2.  **`appsettings.Development.json`'u yapılandırın** yerel veritabanı bağlantı dizeniz ve diğer ayarlarınızla.
3.  **Veritabanı migrasyonlarını uygulayın:**
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```
4  **Veritabanını doldurun** yerel veritabanınıza karşı `Populate_RssSources_Categories.sql` betiğini çalıştırarak.
5.  **APIyi çalıştırın:**
    ```bash
    dotnet run --project WebApi
    ```

Daha kapsamlı detaylar ve üretim dağıtım talimatları için lütfen [özel INSTALL.md rehberine](https://github.com/Opselon/ForexTradingBot/blob/master/WebAPI/INSTALL.md) danışın.

## 🛠️ Geliştirici Rehberi

Bu bölüm geliştirme için yaygın komutları içerir.

### Veritabanı Migrasyonlarını Yönetme

Bu komutları çalıştırmadan önce EF Core araçlarının kurulu olduğundan emin olun: `dotnet tool install --global dotnet-ef`

*   **Yeni migrasyon ekleyin:** Bir domain modelini değiştirdiğinizde, yeni bir migrasyon oluşturun.
    ```bash
    dotnet ef migrations add YourMigrationName --startup-project WebApi --project Infrastructure
    ```
    *(`YourMigrationName`'i açıklayıcı bir isimle değiştirin, örn. `AddSignalStatus`)*

*   **Migrasyonları uygulayın:** Veritabanı şemasını manuel olarak güncellemek için.
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```

### Üretim Derlemesi Oluşturma

Uygulamayı dağıtım için bağımsız bir yürütülebilir dosyaya derlemek için:

```bash
# Windows x64bağımsız derleme örneği
dotnet publish --configuration Release --runtime win-x64elf-contained true --project WebApi
```
*   Çıktı `WebApi/bin/Release/net94ublish` klasöründe olacak.

### Adım 4 Web Kurulum Sihirbazı ✨
Bu yeni, basitleştirilmiş kurulum sürecidir.

1 **Web panelini açın:** Tarayıcınızda **[http://localhost:50ogin.html](http://localhost:50* adresine gidin.2  **Giriş yapın:** Varsayılan kimlik bilgilerini kullanın:
    *   **Kullanıcı adı:** `admin`
    *   **Şifre:** `admin`
    *(Daha iyi güvenlik için web UI'daki ilk kurulum süreci sırasında bu hassas detayları değiştirmeniz istenecektir.)*
3**Rehberli kurulum:** İlk kez giriş yaptıktan sonra otomatik olarak güvenli bir kurulum sayfasına (`/indexapp.html`) yönlendirileceksiniz.
    *   Bu sayfada **Telegram Bot Token**'ınızı ve diğer temel ayarları girmeniz istenecek.
    *   Sistem kimlik bilgilerinizi **canlı olarak test edecek** kaydetmeden önce geçerli olduklarından emin olmak için.
    *   Kaydedildikten sonra bu ayarlar güvenli bir şekilde veritabanında saklanır, düz metin dosyalarında değil.
4  **Veritabanını doldurun:** İlk kurulumdan sonra veritabanını doldurmanız istenecek. Web UIdakiSeed Databasedüğmesine tıklayın. Bu, RSS beslemelerinin başlangıç listesini ve diğer gerekli verileri dolduracak.
