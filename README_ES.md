## 📋 Descripción del proyecto

**ForexSignalBot** es un sistema avanzado basado en IA para Telegram, diseñado para proporcionar señales de trading precisas en tiempo real para el mercado Forex. Construido sobre **.NET 9** utilizando principios de arquitectura limpia y Domain-Driven Design.

### 🌟 Características principales

- **🤖 Generación de señales con IA:** Algoritmos avanzados de análisis de mercado
- **📰 Agregación de noticias:** 100es RSS con categorización inteligente
- **💬 UI completa en Telegram:** Experiencia de usuario rica e interactiva
- **🔗 Reenvío automático:** Integración con plataformas de trading
- **🐳 Contenedorización Docker:** Despliegue fácil y consistente
- **🛡️ Seguridad:** Manejo robusto de errores y protección de datos

### 🏗️ Arquitectura del sistema

El proyecto está construido sobre principios de **arquitectura limpia** y **Domain-Driven Design**:

- **Capa Domain:** Lógica de negocio central y entidades
- **Capa Application:** Servicios de aplicación y orquestación de casos de uso
- **Capa Infrastructure:** Implementación de base de datos y APIs externas
- **WebAPI:** Controladores REST API
- **TelegramPanel:** Gestión de interacciones del bot de Telegram
- **BackgroundTasks:** Procesamiento de tareas en segundo plano con Hangfire

### 🛠️ Stack tecnológico

- **.NET9 - Plataforma principal de desarrollo
- **PostgreSQL** - Base de datos relacional
- **Redis** - Caché y procesamiento en segundo plano
- **Hangfire** - Gestión de tareas en segundo plano
- **Docker** - Contenedorización y despliegue
- **Entity Framework Core** - ORM
- **Polly** - Manejo de errores y resiliencia

## 🚀 Inicio rápido

Puedes ejecutar este proyecto de dos maneras: con Docker (recomendado para configuración rápida) o configurando un entorno local manualmente.

### Opción 1: Inicio rápido con Docker (Recomendado)

Obtén toda la pila de aplicaciones — API, base de datos PostgreSQL y caché Redis — ejecutándose en minutos con Docker. **Esta es la forma más rápida y fácil de comenzar.**

#### Prerrequisitos

*   **Docker Desktop**: Asegúrate de que esté instalado y ejecutándose en tu sistema. [Descárgalo aquí](https://www.docker.com/products/docker-desktop/).

#### Paso 1lonar el repositorio

Abre tu terminal y clona el código fuente del proyecto.
```bash
git clone https://github.com/Opselon/ForexTradingBot.git
cd ForexTradingBot
```

#### Paso2figurar tus secretos

La aplicación requiere claves API y contraseñas. Usamos un archivo `.env` para esto, que se mantiene privado.

1.  **Crear el archivo de entorno:**
    ```bash
    cp .env.example .env
    ```
2.  **Editar el archivo `.env`:** Abre el nuevo archivo `.env` y llena tus valores secretos reales.
    *   `TELEGRAM_BOT_TOKEN`: Obtén esto de `@BotFather` en Telegram.
    *   `POSTGRES_PASSWORD`: Crea una contraseña fuerte y segura para tu base de datos.

#### Paso 3: ¡Ejecutar la aplicación! 🔥

Con Docker ejecutándose, ejecuta un solo comando desde el directorio raíz del proyecto:
```bash
docker-compose up --build -d
```
Este comando construye e inicia los contenedores API, PostgreSQL y Redis. La API está configurada para **aplicar automáticamente las migraciones de la base de datos al iniciar**.

#### Paso4 Poblar la base de datos
El bot necesita una lista inicial de fuentes RSS. Conéctate a la base de datos usando un cliente como DBeaver o DataGrip y ejecuta el script `Populate_RssSources_Categories.sql`.
*   **Host:** `localhost`
*   **Puerto:** `5432   **Base de datos:** `forexsignalbot_db`
*   **Usuario:** `postgres`
*   **Contraseña:** El `POSTGRES_PASSWORD` que configuraste en `.env`.

**🎉 ¡Eso es todo! Tu bot ahora está ejecutándose dentro de Docker.**

### Opción 2: Configuración de desarrollo local (Sin Docker)

Sigue estos pasos si prefieres ejecutar la aplicación directamente en tu máquina.

#### Prerrequisitos
1**.NET9 SDK:**
    *   Instala el **.NET9 SDK (v9107 posterior)**.
    *   **Página de descarga:** [https://dotnet.microsoft.com/en-us/download/dotnet/90//dotnet.microsoft.com/en-us/download/dotnet/9   *   Verifica tu instalación ejecutando `dotnet --version`.
2.  **Base de datos PostgreSQL:**
    *   Instala y ejecuta un servidor PostgreSQL local.
    *   Crea una base de datos y un usuario.
    *   Actualiza tu cadena de conexión en el archivo `appsettings.Development.json`.
3.  **Servidor Redis:**
    *   Redis se usa para caché y procesamiento de trabajos en segundo plano.
    *   **Para Windows:** Instala un servidor compatible con Redis como **Memurai**.
        *   **Guía de instalación:** [https://docs.memurai.com/en/installation.html](https://docs.memurai.com/en/installation.html)
    *   **Para macOS/Linux:** Instala a través de un gestor de paquetes (ej., `brew install redis` o `sudo apt-get install redis-server`).

#### Ejecutar la aplicación localmente

Para desarrolladores que prefieren ejecutar la aplicación directamente en su máquina, sigue estos pasos:

1.  **Clona el repositorio** (si aún no lo has hecho).2  **Configura `appsettings.Development.json`** con tu cadena de conexión de base de datos local y otras configuraciones.
3.  **Aplica las migraciones de la base de datos:**
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```
4.  **Puebla la base de datos** ejecutando el script `Populate_RssSources_Categories.sql` contra tu base de datos local.
5.  **Ejecuta la API:**
    ```bash
    dotnet run --project WebApi
    ```

Para detalles más completos e instrucciones de despliegue en producción, consulta la [guía INSTALL.md dedicada](https://github.com/Opselon/ForexTradingBot/blob/master/WebAPI/INSTALL.md).

## 🛠️ Guía del desarrollador

Esta sección contiene comandos comunes para desarrollo.

### Gestionar migraciones de base de datos

Antes de ejecutar estos comandos, asegúrate de tener las herramientas EF Core instaladas: `dotnet tool install --global dotnet-ef`

*   **Agregar una nueva migración:** Cuando cambies un modelo de dominio, crea una nueva migración.
    ```bash
    dotnet ef migrations add YourMigrationName --startup-project WebApi --project Infrastructure
    ```
    *(Reemplaza `YourMigrationName` con un nombre descriptivo, ej., `AddSignalStatus`)*

*   **Aplicar migraciones:** Para actualizar manualmente el esquema de la base de datos.
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```

### Crear una build de producción

Para compilar la aplicación en un ejecutable autocontenido para despliegue:

```bash
# Ejemplo para una build autocontenida Windows x64
dotnet publish --configuration Release --runtime win-x64elf-contained true --project WebApi
```
*   La salida estará en la carpeta `WebApi/bin/Release/net94ublish`.

### Paso 4tente de configuración web ✨
Este es el nuevo proceso de configuración simplificado.

1.  **Abre el panel web:** Navega a **[http://localhost:50ogin.html](http://localhost:500html)** en tu navegador.
2.  **Inicia sesión:** Usa las credenciales por defecto:
    *   **Usuario:** `admin`
    *   **Contraseña:** `admin`
    *(Se te pedirá cambiar estos detalles sensibles durante el proceso de configuración inicial en la UI web para mayor seguridad.)*
3.  **Configuración guiada:** Después de iniciar sesión por primera vez, serás redirigido automáticamente a una página de configuración segura (`/indexapp.html`).
    *   En esta página se te pedirá ingresar tu **Token del Bot de Telegram** y otras configuraciones principales.
    *   El sistema **probará en vivo** tus credenciales para asegurar que sean válidas antes de guardar.
    *   Una vez guardadas, estas configuraciones se almacenan de forma segura en la base de datos, no en archivos de texto plano.
4.  **Poblar la base de datos:** Después de la configuración inicial, se te pedirá poblar la base de datos. Haz clic en el botónSeed Database" en la UI web. Esto poblará la lista inicial de fuentes RSS y otros datos requeridos.

**🎉 ¡Eso es todo! Tu bot ahora está completamente configurado y ejecutándose.** Puedes gestionar todo desde el panel web.
