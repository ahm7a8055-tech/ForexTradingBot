## 📋 Visão geral do projeto

**ForexSignalBot** é um sistema avançado baseado em IA para Telegram, projetado para fornecer sinais de negociação precisos em tempo real para o mercado Forex. Construído sobre **.NET 9** usando princípios de arquitetura limpa e Domain-Driven Design.

### 🌟 Características principais

- **🤖 Geração de sinais com IA:** Algoritmos avançados de análise de mercado
- **📰 Agregação de notícias:** 100ds RSS com categorização inteligente
- **💬 UI completa no Telegram:** Experiência de usuário rica e interativa
- **🔗 Transmissão automática:** Integração com plataformas de negociação
- **🐳 Containerização Docker:** Implantação fácil e consistente
- **🛡️ Segurança:** Gerenciamento robusto de erros e proteção de dados

### 🏗️ Arquitetura do sistema

O projeto é construído sobre princípios de **arquitetura limpa** e **Domain-Driven Design**:

- **Camada Domain:** Lógica de negócios central e entidades
- **Camada Application:** Serviços de aplicação e orquestração de casos de uso
- **Camada Infrastructure:** Implementação de banco de dados e APIs externas
- **WebAPI:** Controladores REST API
- **TelegramPanel:** Gerenciamento de interações do bot Telegram
- **BackgroundTasks:** Processamento de tarefas em segundo plano com Hangfire

### 🛠️ Stack tecnológico

- **.NET9 - Plataforma principal de desenvolvimento
- **PostgreSQL** - Banco de dados relacional
- **Redis** - Cache e processamento em segundo plano
- **Hangfire** - Gerenciamento de tarefas em segundo plano
- **Docker** - Containerização e implantação
- **Entity Framework Core** - ORM
- **Polly** - Gerenciamento de erros e resiliência

## 🚀 Início rápido

Você pode executar este projeto de duas maneiras: com Docker (recomendado para configuração rápida) ou configurando um ambiente local manualmente.

### Opção 1: Início rápido com Docker (Recomendado)

Obtenha toda a pilha de aplicações — API, banco de dados PostgreSQL e cache Redis — funcionando em minutos com Docker. **Esta é a forma mais rápida e fácil de começar.**

#### Pré-requisitos

*   **Docker Desktop**: Certifique-se de que está instalado e rodando no seu sistema. [Baixe aqui](https://www.docker.com/products/docker-desktop/).

#### Passo 1lone o repositório

Abra seu terminal e clone o código fonte do projeto.
```bash
git clone https://github.com/Opselon/ForexTradingBot.git
cd ForexTradingBot
```

#### Passo2: Configure seus segredos

A aplicação requer chaves API e senhas. Usamos um arquivo `.env` para isso, que é mantido privado.
1  **Crie o arquivo de ambiente:**
    ```bash
    cp .env.example .env
    ```
2 **Edite o arquivo `.env`:** Abra o novo arquivo `.env` e preencha seus valores secretos reais.
    *   `TELEGRAM_BOT_TOKEN`: Obtenha isso do `@BotFather` no Telegram.
    *   `POSTGRES_PASSWORD`: Crie uma senha forte e segura para seu banco de dados.

#### Passo 3xecute a aplicação! 🔥

Com Docker rodando, execute um único comando do diretório raiz do projeto:
```bash
docker-compose up --build -d
```
Este comando constrói e inicia os containers API, PostgreSQL e Redis. A API está configurada para **aplicar automaticamente as migrações do banco de dados na inicialização**.

#### Passo4 Popule o banco de dados
O bot precisa de uma lista inicial de feeds RSS. Conecte-se ao banco de dados usando um cliente como DBeaver ou DataGrip e execute o script `Populate_RssSources_Categories.sql`.
*   **Host:** `localhost`
*   **Porta:** `5432   **Banco de dados:** `forexsignalbot_db`
*   **Usuário:** `postgres`
*   **Senha:** O `POSTGRES_PASSWORD` que você configurou em `.env`.

**🎉 É isso! Seu bot agora está rodando dentro do Docker.**

### Opção 2de desenvolvimento local (Sem Docker)

Siga estes passos se preferir executar a aplicação diretamente na sua máquina.

#### Pré-requisitos
1**.NET9 SDK:**
    *   Instale o **.NET9 SDK (v917 ou posterior)**.
    *   **Página de download:** [https://dotnet.microsoft.com/en-us/download/dotnet/90//dotnet.microsoft.com/en-us/download/dotnet/9Verifique sua instalação executando `dotnet --version`.
2.  **Banco de dados PostgreSQL:**
    *   Instale e execute um servidor PostgreSQL local.
    *   Crie um banco de dados e um usuário.
    *   Atualize sua string de conexão no arquivo `appsettings.Development.json`.
3.  **Servidor Redis:**
    *   Redis é usado para cache e processamento de jobs em segundo plano.
    *   **Para Windows:** Instale um servidor compatível com Redis como **Memurai**.
        *   **Guia de instalação:** [https://docs.memurai.com/en/installation.html](https://docs.memurai.com/en/installation.html)
    *   **Para macOS/Linux:** Instale através de um gerenciador de pacotes (ex., `brew install redis` ou `sudo apt-get install redis-server`).

#### Execute a aplicação localmente

Para desenvolvedores que preferem executar a aplicação diretamente na sua máquina, siga estes passos:
1  **Clone o repositório** (se ainda não fez).2  **Configure `appsettings.Development.json`** com sua string de conexão de banco de dados local e outras configurações.
3.  **Aplique as migrações do banco de dados:**
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```
4  **Popule o banco de dados** executando o script `Populate_RssSources_Categories.sql` contra seu banco de dados local.
5.  **Execute a API:**
    ```bash
    dotnet run --project WebApi
    ```

Para detalhes mais abrangentes e instruções de implantação em produção, consulte o [guia INSTALL.md dedicado](https://github.com/Opselon/ForexTradingBot/blob/master/WebAPI/INSTALL.md).

## 🛠️ Guia do desenvolvedor

Esta seção contém comandos comuns para desenvolvimento.

### Gerenciar migrações de banco de dados

Antes de executar estes comandos, certifique-se de ter as ferramentas EF Core instaladas: `dotnet tool install --global dotnet-ef`

*   **Adicionar uma nova migração:** Quando você alterar um modelo de domínio, crie uma nova migração.
    ```bash
    dotnet ef migrations add YourMigrationName --startup-project WebApi --project Infrastructure
    ```
    *(Substitua `YourMigrationName` por um nome descritivo, ex., `AddSignalStatus`)*

*   **Aplicar migrações:** Para atualizar manualmente o esquema do banco de dados.
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```

### Criar uma build de produção

Para compilar a aplicação em um executável autocontido para implantação:

```bash
# Exemplo para uma build autocontida Windows x64
dotnet publish --configuration Release --runtime win-x64elf-contained true --project WebApi
```
*   A saída estará na pasta `WebApi/bin/Release/net94ublish`.

### Passo 4: Assistente de configuração web ✨
Este é o novo processo de configuração simplificado.
1.  **Abra o painel web:** Navegue para **[http://localhost:50ogin.html](http://localhost:50no seu navegador.
2.  **Faça login:** Use as credenciais padrão:
    *   **Usuário:** `admin`
    *   **Senha:** `admin`
    *(Você será solicitado a alterar estes detalhes sensíveis durante o processo de configuração inicial na UI web para melhor segurança.)*
3 **Configuração guiada:** Após fazer login pela primeira vez, você será automaticamente redirecionado para uma página de configuração segura (`/indexapp.html`).
    *   Nesta página você será solicitado a inserir seu **Token do Bot Telegram** e outras configurações principais.
    *   O sistema **testará em tempo real** suas credenciais para garantir que sejam válidas antes de salvar.
    *   Após salvar, estas configurações são armazenadas com segurança no banco de dados, não em arquivos de texto.
4  **Popule o banco de dados:** Após a configuração inicial, você será solicitado a popular o banco de dados. Clique no botãoSeed Database" na UI web. Isso populará a lista inicial de feeds RSS e outros dados necessários.

**🎉 É isso! Seu bot agora está completamente configurado e funcionando.** Você pode gerenciar tudo do painel web.
