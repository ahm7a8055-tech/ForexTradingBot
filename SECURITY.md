# Security Policy for ForexSignalBot 🛡️

The security of ForexSignalBot is a top priority for us. We are committed to ensuring our platform is secure and reliable for all users. We value the contributions of the security research community and believe that responsible disclosure is the best way to protect our users.

## ✅ Supported Versions

Our development is fast-paced, and we focus our efforts on the latest version of the software. Security patches will only be applied to the most recent stable release or the `master` branch. We encourage all users to stay on the latest version to benefit from the latest features, bug fixes, and security updates.

| Version       | Supported          |
| ------------- | ------------------ |
| `master` branch | :white_check_mark: |
| All older versions | :x:                |

##  रिपोर्टिंग a Vulnerability

We take all security reports seriously. If you believe you have discovered a security vulnerability in ForexSignalBot, we ask that you help us by responsibly disclosing it to us in a private manner.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please use one of the following methods:

1.  **GitHub Private Vulnerability Reporting (Recommended):** Use the [private vulnerability reporting feature](https://github.com/Opselon/ForexTradingBot/security/advisories/new) on GitHub. This is the most secure and efficient way to report a vulnerability.
2.  **Email:** If you prefer, you can send an email to our security team at `security@opselon.com` with the subject line "Security Vulnerability Report: ForexSignalBot".

### What to Include in Your Report

To help us resolve the issue as quickly as possible, please provide a detailed report including:

*   **A clear description** of the vulnerability and its potential impact.
*   **The component or location** where the vulnerability exists (e.g., `WebApi`, `TelegramPanel`, a specific function).
*   **Step-by-step instructions** to reproduce the issue, including any necessary code snippets, scripts, or configurations.
*   **Any proof-of-concept (PoC)** code that demonstrates the vulnerability.
*   **Your contact information** so we can follow up with you.

### Our Commitment and What to Expect

When you report a vulnerability to us, you can expect the following:

1.  **Acknowledgement:** We will acknowledge receipt of your report within **48 hours**.
2.  **Initial Assessment:** We will provide an initial assessment of the report and its severity within **5 business days**.
3.  **Updates:** We will maintain an open line of communication and provide you with periodic updates on our progress as we work to validate and fix the vulnerability.
4.  **Resolution:** We will notify you when the vulnerability has been patched. We aim to release a fix as quickly as possible, depending on the complexity of the issue.
5.  **Public Disclosure & Credit:** Once the fix is deployed, we are happy to publicly credit you for your discovery, unless you prefer to remain anonymous. We will coordinate with you on the timing and content of the public disclosure.

We are incredibly grateful for your help in keeping ForexSignalBot secure and reliable for everyone. Thank you for your contribution!
