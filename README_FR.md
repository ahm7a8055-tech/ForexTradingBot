## 📋 Aperçu du projet

**ForexSignalBot** est un système avancé basé sur lIApour Telegram, conçu pour fournir des signaux de trading précis en temps réel pour le marché Forex. Construit sur **.NET 9** en utilisant les principes d'architecture propre et Domain-Driven Design.

### 🌟 Fonctionnalités principales

- **🤖 Génération de signaux IA:** Algorithmes avancés danalyse de marché
- **📰 Agrégation de nouvelles:** 100s RSS avec catégorisation intelligente
- **💬 UI complète Telegram:** Expérience utilisateur riche et interactive
- **🔗 Transfert automatique:** Intégration avec les plateformes de trading
- **🐳 Conteneurisation Docker:** Déploiement facile et cohérent
- **🛡️ Sécurité:** Gestion robuste des erreurs et protection des données

### 🏗️ Architecture du système

Le projet est construit sur les principes d'**architecture propre** et **Domain-Driven Design**:

- **Couche Domain:** Logique métier centrale et entités
- **Couche Application:** Services d'application et orchestration des cas d'usage
- **Couche Infrastructure:** Implémentation de base de données et APIs externes
- **WebAPI:** Contrôleurs REST API
- **TelegramPanel:** Gestion des interactions du bot Telegram
- **BackgroundTasks:** Traitement des tâches en arrière-plan avec Hangfire

### 🛠️ Stack technologique

- **.NET9 - Plateforme de développement principale
- **PostgreSQL** - Base de données relationnelle
- **Redis** - Cache et traitement en arrière-plan
- **Hangfire** - Gestion des tâches en arrière-plan
- **Docker** - Conteneurisation et déploiement
- **Entity Framework Core** - ORM
- **Polly** - Gestion des erreurs et résilience

## 🚀 Démarrage rapide

Vous pouvez exécuter ce projet de deux manières : avec Docker (recommandé pour une configuration rapide) ou en configurant un environnement local manuellement.

### Option1: Démarrage rapide avec Docker (Recommandé)

Obtenez toute la pile d'applications — API, base de données PostgreSQL et cache Redis — en cours d'exécution en minutes avec Docker. **C'est la façon la plus rapide et la plus facile de commencer.**

#### Prérequis

*   **Docker Desktop** : Assurez-vous qu'il est installé et en cours d'exécution sur votre système. [Téléchargez-le ici](https://www.docker.com/products/docker-desktop/).

#### Étape1 : Cloner le dépôt

Ouvrez votre terminal et clonez le code source du projet.
```bash
git clone https://github.com/Opselon/ForexTradingBot.git
cd ForexTradingBot
```

#### Étape 2igurer vos secrets

L'application nécessite des clés API et des mots de passe. Nous utilisons un fichier `.env` pour cela, qui est gardé privé.

1.  **Créer le fichier d'environnement :**
    ```bash
    cp .env.example .env
    ```2**Modifier le fichier `.env` :** Ouvrez le nouveau fichier `.env` et remplissez vos vraies valeurs secrètes.
    *   `TELEGRAM_BOT_TOKEN` : Obtenez ceci de `@BotFather` sur Telegram.
    *   `POSTGRES_PASSWORD` : Créez un mot de passe fort et sécurisé pour votre base de données.

#### Étape 3 : Exécuter l'application ! 🔥

Avec Docker en cours d'exécution, exécutez une seule commande depuis le répertoire racine du projet :
```bash
docker-compose up --build -d
```
Cette commande construit et démarre les conteneurs API, PostgreSQL et Redis. L'API est configurée pour **appliquer automatiquement les migrations de base de données au démarrage**.

#### Étape 4 : Peupler la base de données
Le bot a besoin dune liste initiale de flux RSS. Connectez-vous à la base de données en utilisant un client comme DBeaver ou DataGrip et exécutez le script `Populate_RssSources_Categories.sql`.
*   **Hôte :** `localhost`
*   **Port :** `5432   **Base de données :** `forexsignalbot_db`
*   **Utilisateur :** `postgres`
*   **Mot de passe :** Le `POSTGRES_PASSWORD` que vous avez défini dans `.env`.

**🎉 C'est tout ! Votre bot fonctionne maintenant dans Docker.**

### Option2: Configuration de développement local (Sans Docker)

Suivez ces étapes si vous préférez exécuter l'application directement sur votre machine.

#### Prérequis1*.NET9 SDK :**
    *   Installez le **.NET9 SDK (v9107 ou plus récent)**.
    *   **Page de téléchargement :** [https://dotnet.microsoft.com/en-us/download/dotnet/90//dotnet.microsoft.com/en-us/download/dotnet/9   *   Vérifiez votre installation en exécutant `dotnet --version`.
2.  **Base de données PostgreSQL :**
    *   Installez et exécutez un serveur PostgreSQL local.
    *   Créez une base de données et un utilisateur.
    *   Mettez à jour votre chaîne de connexion dans le fichier `appsettings.Development.json`.
3.  **Serveur Redis :**
    *   Redis est utilisé pour la mise en cache et le traitement des tâches en arrière-plan.
    *   **Pour Windows :** Installez un serveur compatible Redis comme **Memurai**.
        *   **Guide d'installation :** [https://docs.memurai.com/en/installation.html](https://docs.memurai.com/en/installation.html)
    *   **Pour macOS/Linux :** Installez via un gestionnaire de paquets (ex., `brew install redis` ou `sudo apt-get install redis-server`).

#### Exécuter lapplication localement

Pour les développeurs qui préfèrent exécuter l'application directement sur leur machine, suivez ces étapes :

1.  **Clonez le dépôt** (si vous ne l'avez pas déjà fait).
2 **Configurez `appsettings.Development.json`** avec votre chaîne de connexion de base de données locale et dautres paramètres.
3.  **Appliquez les migrations de base de données :**
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```
4.  **Peuplez la base de données** en exécutant le script `Populate_RssSources_Categories.sql` contre votre base de données locale.5  **Exécutez l'API :**
    ```bash
    dotnet run --project WebApi
    ```

Pour des détails plus complets et des instructions de déploiement en production, veuillez consulter le [guide INSTALL.md dédié](https://github.com/Opselon/ForexTradingBot/blob/master/WebAPI/INSTALL.md).

## 🛠️ Guide du développeur

Cette section contient des commandes communes pour le développement.

### Gérer les migrations de base de données

Avant d'exécuter ces commandes, assurez-vous d'avoir les outils EF Core installés : `dotnet tool install --global dotnet-ef`

*   **Ajouter une nouvelle migration :** Lorsque vous modifiez un modèle de domaine, créez une nouvelle migration.
    ```bash
    dotnet ef migrations add YourMigrationName --startup-project WebApi --project Infrastructure
    ```
    *(Remplacez `YourMigrationName` par un nom descriptif, ex., `AddSignalStatus`)*

*   **Appliquer les migrations :** Pour mettre à jour manuellement le schéma de la base de données.
    ```bash
    dotnet ef database update --startup-project WebApi --project Infrastructure
    ```

### Créer une build de production

Pour compiler lapplication en un exécutable autonome pour le déploiement :

```bash
# Exemple pour une build autonome Windows x64
dotnet publish --configuration Release --runtime win-x64elf-contained true --project WebApi
```
*   La sortie sera dans le dossier `WebApi/bin/Release/net94ublish`.

### Étape 4t de configuration web ✨
Ceci est le nouveau processus de configuration simplifié.

1.  **Ouvrez le panneau web :** Naviguez vers **[http://localhost:50ogin.html](http://localhost:500html)** dans votre navigateur.2.  **Connexion :** Utilisez les identifiants par défaut :
    *   **Nom dutilisateur :** `admin`
    *   **Mot de passe :** `admin`
    *(Il vous sera demandé de changer ces détails sensibles pendant le processus de configuration initial dans l'interface web pour une meilleure sécurité.)*
3.  **Configuration guidée :** Après vous être connecté pour la première fois, vous serez automatiquement redirigé vers une page de configuration sécurisée (`/indexapp.html`).
    *   Sur cette page, il vous sera demandé d'entrer votre **Token du Bot Telegram** et dautresparamètres principaux.
    *   Le système **testera en direct** vos identifiants pour s'assurer qu'ils sont valides avant de sauvegarder.
    *   Une fois sauvegardés, ces paramètres sont stockés en toute sécurité dans la base de données, pas dans des fichiers texte.
4.  **Peupler la base de données :** Après la configuration initiale, il vous sera demandé de peupler la base de données. Cliquez sur le boutonSeed Database dans l'interface web. Cela peuplera la liste initiale des flux RSS et autres données requises.

**🎉 C'est tout ! Votre bot est maintenant complètement configuré et fonctionne.** Vous pouvez tout gérer depuis le panneau web.
